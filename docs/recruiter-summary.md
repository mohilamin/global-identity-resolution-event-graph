# Recruiter Summary

This project shows data engineering work behind large consumer platforms. It solves the problem of fragmented identity: one real user can appear as many cookies, devices, sessions, emails, payments, support accounts, and ad interactions.

The project proves skills in Python, event data modeling, graph analytics, attribution logic, confidence scoring, fraud-aware clustering, API development, dashboarding, testing, Docker, and CI.

It is stronger than a basic ETL project because it models a real platform problem: deciding whether data belongs to the same identity and explaining the decision.
